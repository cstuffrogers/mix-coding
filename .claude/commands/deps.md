# /deps — 依赖更新

## 触发方式
```
> /deps
```

## 执行流程
1. 拉取最新代码
2. 扫描所有过期依赖（自动检测npm/pip/cargo/go mod）
3. 安全审计当前依赖
4. 选择更新策略（安全更新/补丁更新/小版本更新/全量更新）
5. 更新前运行测试建立基线
6. 逐项更新依赖（每次更新后运行测试）
7. 全量回归测试
8. 检查breaking changes
9. 冒烟测试
10. 提交更新的lockfile
11. 通知完成

## 说明
安全依赖更新流程，逐项更新+测试验证，避免一次性全量更新导致的破坏。支持npm/pip/cargo/go mod自动检测。
